How to create a new program

Page 1

  • Log In

    Visit https://admin.bcycle.com - then log in using your admin credentials. On the next screen, just select any program.

  • Enter in program info

    Click 'Programs' (#1), then 'Add new record' (#2), and you should see the popup (#3). Below should be screenshots of the entire program entry - taken in case it changes between the time of this authoring and your reading it.

    Explanation of what to enter

    Note: An asterisk means the field is required. Also note that not all data will be known in advance. If required data is missing, put in fake data in order to save the program. The admins are aware of this.

    1. Contact info - Administrative person in charge of the city and address information which used to be displayed on the dotnetnuke site under 'Contact Us'. In sitefinity however, it's just an editable content block. Name, Street, and City are the only required fields out of this section.
    2. Lock ID - Leave blank for now, honestly not sure what it's for.
    3. *Merchant Name & Transaction Key - use fake data until the correct values are known(FakeName//123FakeKey).
    4. Use Test Server - Charges/refunds will be sent to the auth net test server and not their live system.
    5. Program Group - Unused. leave blank.
    6. *Domain Name - self explanatory.
    7. Website Header Image - Not used in sitefinity.
    8. Kiosk Time Zone / Display Of Units / Culture - self explanatory.
    9. *Overdue Bike Threshold / Bike Inactivity Threshold / Approaching Membership Expiration Threshold - Self Explanatory
    10. Program Model - Should be on initial questionnaire (whether b-cycle uses that questionnaire internally is another matter).
    11. Walkup Purchase Limit - self explanatory.
    12. Public Text - Text that gets shown on the following page (when hovering over the pin I believe): http://sitefinity.bcycle.com/city-locations/
    13. Active - A flag in tblProgram which should indicate whether the program is active. This does affect whether the program is shown in the DNN dropdown. Not sure if it was used in Sitefinity.
    14. Allow User Signup at Kiosk / Wi-MM GPS - Self explanatory.
    15. Open System Up - Restores all kiosks to their regular operating schedule.
    16. Shut System Down - Shuts down all kiosks in the program. During this time, the kiosks will not be operational and only bike returns will be supported.
    17. Show Zip Code / Show Promo Code - Whether or not a zip/promo code is asked for when signing up at a kiosk.
    18. *Map Settings - Settings necessary in order to use the google maps api. In order to get the latitude/longitude of a given point, go to gmaps and left click a point on the map. A box should appear under the search bar with the lat/lon coordinates. If that doesn't work, you're going to have to google how to do it because the method changed.
    19. Default Fees - Leave as is until a city admin chooses to change it.
    20. Grace Period Minutes - Number of minutes after a given half hour before a user is charged for the next half hour overage fee

    Well, you're done. Hit that save button.
